# 日報

## 本日の結果

担当している顧客データ取込機能の結合テストを実施しました。

午前中に予定していた正常系12件はすべて実施し、11件が成功しました。残り1件では、会社名に外字が含まれる場合、取込後に文字化けする事象が発生しました。確認の結果、画面表示だけでなく、CSV取込直後のデータにも代替文字が保存されていました。

原因を調査したところ、取込ライブラリがUTF-8前提である一方、対象CSVは取引先の古いシステムから出力されたWindows-31J形式でした。文字コードを自動推測すると誤変換の可能性があるため、自動判定は採用せず、取込前に文字コードを選択し、不正な文字がある場合は保存前に処理を止める案を作成しました。

15時に高橋さんへ相談し、この案で翌日に試作することになりました。文字化けした試験データは削除済みで、本番データへの影響はありません。

異常系テストは、予定8件のうち3件まで完了しました。

## 明日の対応

- 保存前の文字検査を試作する
