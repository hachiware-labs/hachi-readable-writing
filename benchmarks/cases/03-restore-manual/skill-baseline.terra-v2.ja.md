# 削除したワークスペースを復元する手順

削除から30日以内のワークスペースは、組織所有者が管理画面から復元できます。復元すると、メンバー、設定、ファイルが元に戻ります。

30日を過ぎたワークスペースは復元できません。

## 事前確認

- 作業者が組織所有者であることを確認します。組織管理者でも、組織所有者でない場合は復元できません。
- 削除から30日以内であることを確認します。
- 削除後に同じURL用識別子を別のワークスペースで使用している場合は、先にその現在のワークスペースの識別子を変更します。
- 復元には最大20分かかります。復元中、対象ワークスペースのファイルは読み取り専用になります。

## 復元手順

1. 管理画面に、組織所有者のアカウントでログインします。

2. `Settings` を開き、`Deleted workspaces` に進みます。

3. 復元するワークスペースを選択します。

4. `Review restore` を開き、ワークスペース名とURL用識別子を確認します。

5. 内容に問題がなければ、`Restore workspace` を実行します。

復元処理が完了するまで待ちます。完了までの間は、対象ワークスペースのファイルを変更しないでください。

## 復元後の確認と再開

復元が完了したら、メンバーとファイルが戻っていることを確認します。

削除時に停止した外部連携は、復元後も無効のままです。必要な外部連携を個別に再有効化してください。

また、定期実行ジョブは自動では再開しません。ジョブの内容と実行条件を確認してから、必要なものを再開します。
