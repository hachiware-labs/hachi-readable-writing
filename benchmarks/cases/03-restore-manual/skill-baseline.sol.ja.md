# 削除したワークスペースを復元する

この手順書は、削除後30日以内のワークスペースを管理画面から復元するときに使用します。復元できるのは組織所有者だけです。

復元すると、メンバー、設定、ファイルが戻ります。ただし、削除時に停止した外部連携は無効のままです。また、定期実行ジョブも自動では再開しません。復元後にそれぞれ確認してください。

## 復元前の確認

作業を始める前に、次の条件を確認します。

- 作業者が組織所有者である。
- ワークスペースの削除から30日以内である。30日を過ぎたワークスペースは復元できない。
- 削除後、同じURL用識別子を別のワークスペースで使用していない。

同じURL用識別子が現在の別のワークスペースに割り当てられている場合は、先にそのワークスペースの識別子を変更します。その後、復元作業に進んでください。

## 復元手順

1. 組織所有者のアカウントで管理画面にログインします。

2. `Settings` を開きます。

3. `Deleted workspaces` を開きます。

4. 復元するワークスペースを選びます。削除から30日以内の対象であることを確認してください。

5. `Review restore` を開き、ワークスペースの名前とURL用識別子を確認します。別のワークスペースが同じURL用識別子を使用している場合は、復元を実行せず、先に現在使用している側の識別子を変更します。

6. 内容に問題がなければ、`Restore workspace` を実行します。

復元には最大20分かかります。処理中、復元対象のファイルは読み取り専用になります。復元が完了するまでは、ファイルを更新しないでください。

## 復元後の確認

復元が完了したら、次の順に状態を確認します。

1. メンバーが戻っていることを確認します。

2. 必要なファイルが戻っていることを確認します。

3. 必要な外部連携を個別に再有効化します。削除時に停止した外部連携は、復元しても無効のままです。

4. 定期実行ジョブの内容と状態を確認してから、必要なジョブを再開します。定期実行ジョブは自動では再開しません。

メンバーとファイルが戻り、必要な外部連携と定期実行ジョブを再開できた時点で、復元作業は完了です。
