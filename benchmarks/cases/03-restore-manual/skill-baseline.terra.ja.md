# 削除したワークスペースを復元する手順

この手順は、削除したワークスペースを管理画面から復元する組織所有者向けです。復元できるのは削除後30日以内のワークスペースに限られます。

## 事前に確認すること

復元作業は組織所有者が行ってください。組織管理者であっても、組織所有者でなければ復元できません。

削除から30日を過ぎたワークスペースは復元できません。また、削除後に同じURL用識別子を別のワークスペースで使用している場合は、先に現在のワークスペースの識別子を変更してください。

復元すると、削除前のメンバー、設定、ファイルが戻ります。ただし、削除時に停止した外部連携は無効のままであり、定期実行ジョブも自動では再開しません。

## 復元する

1. 管理画面にログインします。

2. `Settings` を開き、続けて `Deleted workspaces` を開きます。

3. 復元するワークスペースを選択します。

4. `Review restore` を選び、ワークスペース名とURL用識別子を確認します。

5. 内容に問題がなければ、`Restore workspace` を実行します。

復元には最大20分かかります。復元中は、対象ワークスペースのファイルが読み取り専用になります。

## 復元後に行うこと

復元が完了したら、メンバーとファイルが戻っていることを確認します。続いて、業務で必要な外部連携を個別に再有効化してください。

定期実行ジョブは停止したままです。内容と実行状況を確認したうえで、必要なジョブを再開します。
