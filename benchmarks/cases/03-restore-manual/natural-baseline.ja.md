# 削除したワークスペースを復元する手順

## 対象者

組織の管理者向けの手順です。ただし、復元を実行できるのは**組織所有者のみ**です。組織所有者以外は、所有者に作業を依頼してください。

## 復元前の確認

作業を始める前に、次の点を確認します。

- ワークスペースの削除から30日以内であること
  - 削除後30日間は復元できます。
  - 30日を過ぎたワークスペースは復元できません。
- 復元対象のワークスペース名とURL用識別子が分かっていること
- 元のURL用識別子を、削除後に別のワークスペースで使用していないこと
  - 使用している場合は、先に現在のワークスペースのURL用識別子を変更してください。
- 復元に最大20分かかることを関係者に共有していること
  - 復元中、対象のファイルは読み取り専用になります。

## 復元手順

1. 組織所有者のアカウントで管理画面にログインします。
2. `Settings` を開きます。
3. `Deleted workspaces` を開きます。
4. 復元するワークスペースを選択します。
5. `Review restore` を開きます。
6. 表示された次の情報が、復元対象と一致していることを確認します。
   - ワークスペース名
   - URL用識別子
7. 内容に問題がなければ、`Restore workspace` を実行します。
8. 復元が完了するまで待ちます。完了には最大20分かかります。

## 復元後の確認

復元が完了したら、次の順序で確認します。

1. メンバーが戻っていることを確認します。
2. 設定が戻っていることを確認します。
3. ファイルが戻り、読み取り専用ではなくなっていることを確認します。
4. 必要な外部連携を個別に再有効化します。
   - 削除時に停止した外部連携は、復元後も無効のままです。
5. 定期実行ジョブの内容と状態を確認します。
6. 問題がないことを確認してから、必要な定期実行ジョブを再開します。
   - 定期実行ジョブは自動では再開されません。

## 復元されるもの・自動復旧しないもの

| 項目 | 復元後の状態 |
|---|---|
| メンバー | 復元される |
| 設定 | 復元される |
| ファイル | 復元される |
| 削除時に停止した外部連携 | 無効のまま。個別の再有効化が必要 |
| 定期実行ジョブ | 自動再開しない。確認後に再開が必要 |
