# 削除したワークスペースを復元する手順書

## 1. 対象者

この作業は、対象組織の「組織所有者」が実施してください。  
組織の管理者であっても、組織所有者でない場合は復元できません。

## 2. 復元できる期間と復元内容

- ワークスペースの削除後30日間は、論理削除の状態として復元できます。
- 削除から30日を過ぎると復元できません。
- 復元すると、次の内容が戻ります。
  - メンバー
  - 設定
  - ファイル
- 削除時に停止した外部連携は、無効のままです。必要に応じて個別に再有効化してください。
- 定期実行ジョブは自動的に再開しません。

## 3. 事前確認

作業前に、次の点を確認します。

- 自分が組織所有者である
- 対象ワークスペースを削除してから30日以内である
- 復元するワークスペースの名前とURL用識別子に誤りがない
- 同じURL用識別子を、別のワークスペースで使用していない

同じURL用識別子を別のワークスペースで使用している場合は、先に現在のワークスペースのURL用識別子を変更してください。

## 4. 復元手順

1. 管理画面にログインします。
2. `Settings` を開きます。
3. `Deleted workspaces` を開きます。
4. 復元するワークスペースを選択します。
5. `Review restore` を選択します。
6. 表示されたワークスペースの名前とURL用識別子を確認します。
7. 内容に問題がなければ、`Restore workspace` を実行します。
8. 復元処理が完了するまで待ちます。

復元には最大20分かかる場合があります。復元中は、復元対象のファイルが読み取り専用になります。

## 5. 復元完了後の確認

復元処理が完了したら、次の項目を確認します。

- メンバーが戻っている
- 設定が戻っている
- ファイルが戻っている
- 必要な外部連携を再有効化する
- 定期実行ジョブの状態を確認する
- 必要な定期実行ジョブを手動で再開する

外部連携と定期実行ジョブは、復元後に自動では再開しません。利用を再開する前に、必ず状態を確認してください。

## 6. 復元できない場合

次のいずれかに該当する場合、復元できません。

- 作業者が組織所有者ではない
- ワークスペースの削除から30日を過ぎている
- URL用識別子が別のワークスペースで使用されている

URL用識別子が重複している場合は、現在その識別子を使用しているワークスペースの識別子を変更してから、復元を実行してください。
