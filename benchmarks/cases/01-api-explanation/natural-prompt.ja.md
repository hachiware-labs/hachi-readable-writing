Kite Mail APIを初めて組み込む開発者向けに、送信予約の使い方を説明する文書を作ってください。
社内の開発者ポータルに載せます。

背景として分かっていること:

- ベースURLは `https://api.kitemail.example`。
- トランザクションメールの送信予約には `POST /v1/messages` を使う。
- `Authorization: Bearer <token>` と `Idempotency-Key` は必須。
- リクエスト本文には `to`、`template_id`、`variables` を含められる。`send_at` は任意で、RFC 3339形式。1回のリクエストの宛先は100件までで、`send_at` は現在から7日以内に限る。
- 正常時は `202 Accepted` を返し、本文には `message_id` と `status: queued` が含まれる。202は配信完了を意味しない。
- 同じ `Idempotency-Key` と同じ本文を24時間以内に再送すると、最初のレスポンスを返す。同じキーで本文が異なる場合は409。
- 主なエラーは400（入力不正）、401（認証失敗）、409（冪等性キーの競合）、429（送信頻度の上限超過）。429では `Retry-After` ヘッダーに従う。
- 400、401、409は、そのまま自動再試行しない。ネットワークエラーと429は再試行してよい。
- ここに書いた以外の機能やURLは分からない
