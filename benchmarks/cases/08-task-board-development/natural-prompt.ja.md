小さなWebアプリ「Pocket Queue」を実装するための開発ガイドを作ってください。
Reactを使ったことがある開発者向けで、データ型と保存処理のコード例もほしいです。

背景として分かっていること:

- React 19、Vite 7、TypeScriptのstrictモードを使う。
- 端末内だけで動く1画面のタスクボードで、サーバー、認証、同期はない。
- 列は `backlog`、`doing`、`done` の3つ。
- タスクは `id`、`title`、`status`、`createdAt`、`updatedAt` を持つ。`id` はUUID、タイトルは1～80文字。
- 追加、タイトル編集、列の移動、削除、列による絞り込みができる。
- ドラッグ操作だけに依存せず、キーボードでも列を移動できる。
- 保存先はlocalStorageの `pocket-queue:v1`。状態変更後に保存し、起動時に読み込む。
- 保存データがJSONとして壊れている、または必要な形を満たさない場合は、空のボードに戻して画面上に警告を出す。壊れた値を上書きするのは、利用者が次に状態を変更したとき。
- 状態更新はreducerに集約し、localStorageの読み書きと検証は別モジュールに分ける。
- 外部のドラッグ・アンド・ドロップライブラリは使わない。
- VitestとReact Testing Libraryで、reducer、保存データの検証、追加と列移動の利用者操作をテストする。
- このほかの機能は今回の対象にしていない
