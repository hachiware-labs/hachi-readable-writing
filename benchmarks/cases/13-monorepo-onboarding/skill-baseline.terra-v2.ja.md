# Northstar 開発参加ガイド

Northstar は TypeScript のモノレポです。最初は `packages/domain` の期限表示を修正し、テスト付きで提出するところまで進めます。

## セットアップ

Node.js 22 と pnpm 10 を用意してから、リポジトリのルートで依存関係を入れます。

```sh
pnpm install --frozen-lockfile
```

`--frozen-lockfile` は既存のロックファイルに従って依存関係を再現します。依存関係を追加・更新しない限り、ロックファイルは変更しません。

開発時は、次のコマンドで Web と API を同時に起動できます。

```sh
pnpm dev
```

環境変数やポート番号は、現時点では別途案内されていません。起動に必要な値が分からない場合は、推測して追加せず、チームに確認してください。

## 構成を把握する

主な担当範囲は次のとおりです。

| 場所 | 役割 |
| --- | --- |
| `apps/web` | React の画面 |
| `apps/api` | HTTP API |
| `packages/domain` | 業務ルール |
| `packages/ui` | 共有画面部品 |

今回の「期限表示」は業務ルールなので、修正先は `packages/domain` です。同じ判定を `apps/web` や `apps/api` に複製せず、利用側は `packages/domain` の関数を使う形にします。

## 最初の変更

課題は、`packages/domain` にある期限表示関数が期限当日を `due today` と返すようにすることです。

まず既存の関数とテストを読み、期限前・期限後・期限当日の扱いを確認します。期限当日の条件を関数に追加し、既存のテストの書き方に合わせて「期限当日なら `due today` を返す」単体テストを加えます。時刻やタイムゾーンの扱いが既存コードから判断できない場合は、仕様を補わず確認してください。

パッケージ単体のテストは、ルートから実行します。`<package-name>` には `packages/domain/package.json` にある実際のパッケージ名を指定してください。

```sh
pnpm --filter <package-name> test
```

## 提出前の確認

変更後は、まず対象パッケージのテストを通し、続けてリポジトリ全体の型検査と整形確認を行います。

```sh
pnpm typecheck
pnpm format:check
```

提出前には、全体テストも実行します。

```sh
pnpm test
```

すべて通ったら、変更内容を確認します。今回含めるべきものは、期限当日の表示修正と、その振る舞いを守る単体テストです。依存関係を変更していないなら、ロックファイルは変更に含めません。

ブランチ名と提出方法はまだ案内されていないため、チームの運用を確認してからコミットとレビュー依頼を行ってください。
