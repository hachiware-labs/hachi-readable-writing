# Natural baseline editorial evaluation

評価日: 2026-07-12  
対象: `natural-baseline.ja.md`、`natural-baseline.en.md` 全40文書

これは人間の被験者を集めた調査結果ではなく、Codexによる編集評価である。「人間が最初から最後まで無理なく読み、内容と判断の流れを理解できるか」を基準に、全件を同じ観点で通読した。後日、母語話者による盲検比較を行う場合の暫定基準値として扱う。

## 採点基準

各項目25点、合計100点とした。

- **流れ**: 情報の順序に理由があり、前後の段落の関係を追えるか。
- **理解**: 読者が内容、制約、次の行動を誤解なく把握できるか。
- **言葉**: 語彙、文の長さ、調子、専門語が読者と用途に合うか。
- **形式**: 見出し、箇条書き、表、コードが理解を助け、本文の代わりになっていないか。

点数は文章の読みやすさと理解しやすさを対象とする。入力に存在しなかった機能の追加などは、それによって読者が誤解する場合に「理解」から減点した。単に長い、短いという理由だけでは減点せず、その長さが用途に釣り合うかで判断した。

## 全体集計

| 区分 | 平均点 |
|---|---:|
| 日本語30文書 | **84.9** |
| 英語30文書 | **83.6** |
| 技術文書20文書（日英） | **76.9** |
| 日常文書30文書（日英） | **89.9** |
| プレゼン文書10文書（日英） | **82.1** |
| 全60文書 | **84.3** |

平均値だけを見ると日英差は小さい。しかし、問題の現れ方は異なる。日本語の技術文書では、名詞中心の短い項目、同型の見出し、箇条書きの多層化によって、個々の情報は読めても「なぜ次にその話へ進むのか」が弱くなりやすい。英語でも過剰な節立てはあるが、段落内部で因果関係を説明する割合が比較的高い。

最も大きな差は言語間ではなく文書種別間にあった。メール、通知、出品文は宛先と目的が具体的なため、短い散文へ自然に収束した。追加したエッセイ、成長記録、旅行記では、具体的な反復要素や転換点が与えられると、長めの日常文でも高い一貫性を作れた。ただし日報の英語版は情報を見出しと箇条書きへ変換し、同じ材料を使った日本語版より一日の因果的な流れが弱くなった。設計、障害対応、セキュリティー手順、開発ガイドでは、モデルが「抜けなく網羅すること」を優先し、入力以上の詳細、見出し、チェック項目を増やした。その結果、情報量は増えても読者が保持すべき主線は弱くなった。

プレゼン文書では、主張の順序よりも分量調整に大きな問題が出た。日本語版はおおむね指定時間に近い長さへ収まったが、英語版は10～15分の依頼に対して約6,800～11,000字となった。スライドや節ごとの論理は整っていても、表示内容と読み上げ原稿の重複、数字の再掲、過剰なまとめによって、聞き手が時間内に理解できる密度を超えている。プレゼンでは「書いてある内容が理解できるか」だけでなく、「一度聞いて理解できる量か」を独立して制御する必要がある。

この結果から、作成するスキルには単なる箇条書き禁止ではなく、少なくとも次の判断が必要になる。

- 文書の中心となる問いを一つ定め、その問いに沿って情報を選ぶ。
- 見出しを付ける前に、段落同士の因果、対比、時間関係を文章でつなぐ。
- コードやチェックリストは本文で説明した判断を支える範囲に留める。
- 入力にない詳細を補う場合は、既知の事実、例示、提案、未決事項を区別する。
- 緊急時文書では網羅性だけでなく、最初に実行する一本の経路を守る。
- 日本語では名詞句を並べるだけにせず、誰が何をなぜ行うのかを文として示す。
